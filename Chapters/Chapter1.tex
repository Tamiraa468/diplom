\subsection{Судалгааны үндэслэл}
\label{subsec:background}

Мэдээллийн технологийн хурдацтай хөгжил нь бизнесийн бүхий л салбарт, тэр дундаа нийлүүлэлтийн сүлжээний менежментэд (Supply Chain Management --- SCM) томоохон өөрчлөлтүүдийг авчирч байна~\cite{chopra2019supply}. Дэлхий дахинд B2B (Business-to-Business) цахим худалдааны зах зээл жилээс жилд өргөжин тэлж, уламжлалт худалдааны аргуудыг орлож байна. \textit{Grand View Research}-ийн судалгаагаар дэлхийн B2B цахим худалдааны зах зээлийн хэмжээ 2028 он гэхэд 25.6 их наяд ам.долларт хүрэх төлөвтэй байна~\cite{grandview2023}. Энэхүү өсөлт нь бизнесүүдийг үйл ажиллагаагаа цахимжуулах, зардлаа бууруулах, үр ашгаа нэмэгдүүлэх шаардлагатай нүүр тулгаж байна.

\paragraph{Монгол улсын нөхцөл байдал}
Монгол улсын хувьд жижиг, дунд бизнес (ЖДБ) нь эдийн засгийн томоохон хэсгийг бүрдүүлдэг хэдий ч тэдгээрийн бараа татан авалт, түгээлтийн үйл явц нь ихэвчлэн механик, гар ажиллагаатай хэвээр байна. Үндэсний Статистикийн Хорооны мэдээгээр ЖДБ-үүд нийт аж ахуйн нэгжийн 98\%-ийг бүрдүүлдэг~\cite{nso2023}. Улаанбаатар хотын замын түгжрэл, хаягжилтын тодорхойгүй байдал, цаг агаарын эрс тэс уур амьсгал зэрэг нь логистикийн үйл ажиллагаанд нэмэлт хүндрэлүүдийг учруулдаг. Одоогийн байдлаар нийлүүлэгч болон захиалагч (дэлгүүр, ресторан, ТҮЦ гэх мэт) хоорондын харилцаа дараах үндсэн хүндрэлүүдтэй тулгарч байна:

\paragraph{Үндсэн асуудлууд}

\begin{enumerate}
    \item \textbf{Мэдээллийн урсгалын удаашрал ба алдаа:} Захиалгыг утсаар ярих, мессеж бичих, эсвэл худалдааны төлөөлөгч биечлэн очиж авах зэрэг уламжлалт аргуудаар гүйцэтгэж байгаа нь цаг хугацаа их шаарддаг. ``Захиалга мартсан'', ``Буруу бараа ачсан'', ``Тоо хэмжээ зөрсөн'' зэрэг алдаанууд нь хүний хүчин зүйлээс шууд хамааралтай байдаг.
    
    \item \textbf{Бодит цагийн хяналтгүй байдал} \textit{(Lack of Real-time Visibility):} Захиалагч тал бараагаа хэзээ ирэхийг мэддэггүй, нийлүүлэгч тал хүргэгчээ хаана явж байгааг хянаж чаддаггүй. Энэ нь үл ойлголцол үүсгэж, хэрэглэгчийн сэтгэл ханамжийг бууруулдаг.
    
    \item \textbf{Баримт бичгийн эмх замбараагүй байдал:} Цаасан падаан, нэхэмжлэх ашиглах нь баримт бичиг гээгдэх, урагдах, тооцоо нийлэхэд хүндрэл учруулах эрсдэлтэй. Санхүүгийн тооцоог гараар шивэх нь давхар ажил үүсгэдэг.
    
    \item \textbf{Маршрутын оновчлолгүй байдал:} Хүргэгчид өөрсдийн туршлагаар замаа сонгодог нь шатахууны зарцуулалтыг нэмэгдүүлж, хүргэлтийн хугацааг уртасгадаг~\cite{toth2014vehicle}.
\end{enumerate}

Эдгээр асуудлууд нь бизнесийн үйл ажиллагааны зардлыг нэмэгдүүлж, бүтээмжийг бууруулж, эцсийн хэрэглэгчид хүрэх үйлчилгээний чанарт сөргөөр нөлөөлж байна. Иймд орчин үеийн гар утасны технологид суурилсан, автоматжуулсан нэгдсэн системийг хөгжүүлж нэвтрүүлэх нь цаг үеийн зайлшгүй шаардлага болж байна~\cite{wang2016bigdata}.

\subsection{Судалгааны зорилго}
\label{subsec:objective}

Энэхүү дипломын ажлын үндсэн зорилго нь Монгол улсын жижиг, дунд бизнесүүдийн хооронд (B2B) бүтээгдэхүүн захиалах, хүлээн авах, хүргэх үйл явцыг нэгтгэсэн, мэдээллийн технологийн дэвшилтэт шийдэл бүхий \textbf{``Tdelivery''} гар утасны аппликейшн системийг зохион бүтээж, хөгжүүлэхэд оршино.

\noindent Энэхүү систем нь дараах үндсэн функцуудыг хэрэгжүүлнэ:
\begin{itemize}
    \item Захиалгын удирдлагыг автоматжуулах
    \item Хүргэлтийн явцыг бодит цагт хянах
    \item Цаасан баримтыг цахимжуулах
    \item Гурван талт (нийлүүлэгч, захиалагч, хүргэгч) нэгдсэн платформ бий болгох
\end{itemize}

\subsection{Судалгааны зорилтууд}
\label{subsec:tasks}

Дээрх зорилгод хүрэхийн тулд дараах нарийвчилсан зорилтуудыг дэвшүүлж, шийдвэрлэнэ (Хүснэгт~\ref{tab:research_tasks}-д товчилсон байдлаар харуулав):

\begin{enumerate}
    \item \textbf{Онолын болон практик судалгаа хийх:}
    \begin{itemize}
        \item Нийлүүлэлтийн сүлжээний удирдлага, "Last-mile delivery" ойлголтуудыг судлах.
        \item Одоо ашиглагдаж буй дотоодын болон гадаадын ижил төстэй системүүдэд харьцуулсан шинжилгээ хийж, давуу болон сул талуудыг тодорхойлох.
        \item NoSQL өгөгдлийн сангийн бүтэц, React Native фреймворкийн онцлогийг судлах.
    \end{itemize}
    
    \item \textbf{Системийн шинжилгээ ба зохиомж:}
    \begin{itemize}
        \item Хэрэглэгчийн шаардлагыг тодорхойлж, Use Case, Activity, Sequence диаграммуудыг боловсруулах.
        \item Системийн архитектур болон өгөгдлийн сангийн (ERD) зохиомжийг гаргах.
        \item Хэрэглэгчийн интерфэйс (UI/UX) загварыг гаргах.
    \end{itemize}
    
    \item \textbf{Программын хөгжүүлэлт:}
    \begin{itemize}
        \item React Native ашиглан Захиалагч, Нийлүүлэгч, Хүргэгчдэд зориулсан гар утасны аппликейшнийг хөгжүүлэх.
        \item Google Firebase (Authentication, Firestore, Cloud Functions) ашиглан Backend системийг бүрдүүлэх.
        \item Google Maps API ашиглан байршил тогтоох, маршрут харуулах функцуудыг хөгжүүлэх.
    \end{itemize}
    
    \item \textbf{Туршилт ба үнэлгээ:}
    \begin{itemize}
        \item Системийн үйл ажиллагааг тестлэх (Unit test, Integration test).
        \item Бодит өгөгдөл дээр туршилт хийж, үр дүнг үнэлэх.
    \end{itemize}
\end{enumerate}

\subsection{Судалгааны ач холбогдол}
\label{subsec:significance}

Энэхүү судалгааны ажлын хүрээнд хөгжүүлж буй ``Tdelivery'' систем нь дараах практик болон онолын ач холбогдолтой:

\subsubsection{Практик ач холбогдол}
\begin{itemize}
    \item \textbf{Үйл ажиллагааны үр ашиг:} Захиалга, хүргэлтийн процессыг автоматжуулснаар цаг хугацааг 50--70\% хэмнэх, хүний нөөцийн зардлыг бууруулах боломжийг олгоно.
    
    \item \textbf{Мэдээллийн ил тод байдал:} Захиалагч, нийлүүлэгч, хүргэгч гэсэн гурван тал нэгдсэн платформ дээр бодит цагийн \textit{(Real-time)} мэдээлэл солилцох боломжтой болсноор үл ойлголцол арилж, итгэлцэл нэмэгдэнэ.
    
    \item \textbf{Байгаль орчинд ээлтэй:} Цаасан баримт бичгийн хэрэглээг халж, цахим баримт \textit{(e-POD --- electronic Proof of Delivery)} нэвтрүүлснээр байгаль орчинд ээлтэй ``Ногоон логистик''-ийн чиг хандлагад нийцнэ~\cite{greenlogistics2020}.
    
    \item \textbf{Өгөгдөлд суурилсан шийдвэр гаргалт:} Системийн цуглуулсан өгөгдөлд дүн шинжилгээ хийснээр бизнесийн оновчтой шийдвэр гаргах боломжтой болно.
\end{itemize}

\subsubsection{Нийгэм, эдийн засгийн ач холбогдол}
\begin{itemize}
    \item Жижиг дунд бизнес эрхлэгчдийн өрсөлдөх чадварыг нэмэгдүүлнэ.
    \item Хүргэлтийн үйлчилгээний чанарыг сайжруулснаар эцсийн хэрэглэгчийн сэтгэл ханамж өснө.
    \item Мэдээллийн технологийн дэвшлийг уламжлалт салбарт нэвтрүүлж, дижитал шилжилтийг хурдасгана.
\end{itemize}

\subsection{Судалгааны хамрах хүрээ ба хязгаарлалт}
\label{subsec:scope}

\subsubsection{Хамрах хүрээ}
\begin{itemize}
    \item \textbf{Газарзүйн хүрээ:} Улаанбаатар хотын хэмжээнд үйл ажиллагаа явуулж буй хүнсний болон өргөн хэрэглээний барааны нийлүүлэлтийн сүлжээ.
    \item \textbf{Техникийн хүрээ:} iOS болон Android үйлдлийн системтэй ухаалаг гар утас хэрэглэгчид.
    \item \textbf{Тээврийн хүрээ:} Жижиг оврын ачаа тээвэр (мотоцикл, суудлын тэрэг, портер).
    \item \textbf{Хэрэглэгчийн хүрээ:} Нийлүүлэгч, захиалагч (жижиглэн худалдааны цэгүүд), хүргэгч нар.
\end{itemize}

\subsubsection{Хязгаарлалт}
\begin{itemize}
    \item \textbf{Сүлжээний хамаарал:} Систем нь зөвхөн интернэт сүлжээтэй орчинд бүрэн ажиллах боломжтой (хэдийгээр оффлайн горим дэмждэг ч өгөгдөл синхрончлоход сүлжээ шаардлагатай).
    \item \textbf{Төлбөрийн систем:} Төлбөр тооцооны систем нь одоогоор зөвхөн бэлэн мөнгө болон дансны шилжүүлгийг бүртгэх хэлбэрээр шийдэгдсэн (Банкны API шууд холболт хийгдээгүй).
    \item \textbf{Ачааны төрөл:} Хүнд даацын болон тусгай зөвшөөрөл шаардлагатай ачаа тээвэрлэлтийг хамраагүй.
    \item \textbf{Хугацааны хязгаар:} Туршилтын үе шат зөвхөн 2--3 сарын хугацаанд хийгдсэн.
\end{itemize}

\subsection{Дипломын ажлын бүтэц}
\label{subsec:structure}

Энэхүү дипломын ажил нь удиртгал, 4 үндсэн бүлэг, дүгнэлт, ном зүй, хавсралт гэсэн хэсгүүдээс бүрдэнэ (Зураг~\ref{fig:thesis_structure}).

\begin{description}
    \item[Нэгдүгээр бүлэг:] Судалгааны ажлын үндэслэл, зорилго, зорилт, ач холбогдлыг тодорхойлно.
    
    \item[Хоёрдугаар бүлэг:] Сэдвийн хүрээн дэх онолын судалгаа болон ижил төстэй системүүдийн харьцуулсан шинжилгээг хийнэ.
    
    \item[Гуравдугаар бүлэг:] Системийн шинжилгээ, зохиомж, архитектурын шийдлийг дэлгэрэнгүй танилцуулна.
    
    \item[Дөрөвдүгээр бүлэг:] Системийн хэрэгжүүлэлт, ашигласан технологиуд, кодын тайлбар, туршилтын үр дүнг багтаана.
    
    \item[Дүгнэлт:] Ажлын үр дүнг нэгтгэн дүгнэж, цаашдын хөгжүүлэлтийн саналыг дэвшүүлнэ.
\end{description}
